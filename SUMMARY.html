End-of-Turn Detection Assignment

Build a model that predicts when a speaker is done talking.

This project is about end-of-turn detection for a live voice agent. For every pause inside a user turn, the model must estimate the probability that the user is actually finished, while only using audio that has already happened.

What the PDF asks for

The handout describes a real-time endpointing problem: at each pause, decide whether to respond now or keep waiting. The hard rule is causality, so features for a pause may only use audio from the start of the turn up to pause_start. Future audio, including pause_end, is forbidden.

The scorer simulates a live agent and reports mean response delay at a false-cutoff budget of 5%. Lower is better, but the model must keep interruptions under control.

Dataset structure

The archive contains English and Hindi folders. Each has an audio directory with one WAV per turn and a labels.csv file listing pauses within each turn. Each row labels a pause as either hold or eot.

  • turn_id identifies the user turn.
  • audio_file points to the WAV file.
  • pause_index counts the pauses within a turn.
  • pause_start and pause_end define the annotated silence.
  • label tells whether the user continues or ends the turn.

What has been implemented

  • predict.py produces the required submission CSV format.
  • features.py provides causal audio loading, framing, energy, and pitch utilities.
  • modeling.py extracts prosodic features and trains a logistic regression classifier.
  • baseline.py and score.py mirror the starter interface and evaluation logic.
  • RUNLOG.md and NOTES.md track the first validated results and modeling observations.

Why this should beat silence-only

The silence baseline treats every pause as a turn ending. That is simple, but it waits only on silence and ignores whether the speaker is still building the utterance. The current model adds causal cues from the speech immediately before the pause: energy drop, pitch contour, voiced segment length, and other short-term prosodic signals. Those features help distinguish a real terminal pause from a mid-turn hesitation.

The next likely improvements are better pause-position features, more robust voiced-region summaries, and calibration work to tune the false-cutoff tradeoff more aggressively.
